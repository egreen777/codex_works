주식투자 백테스팅 기능 개발
1. 목표를 먼저 정하기
      - 단타/스윙/장기투자 중 무엇인지: 장기 
      - 국내주식/미국주식/ETF 중 무엇인지: 미국 ETF
      - 일봉 기준인지, 분봉까지 할 건지: 일봉
      - 단순 전략 검증용인지, 실전매매 보조용인지: 전략검증 
2. 백테스트 엔진 만들기
      - 핵심은 과거 데이터에 대해 매수/매도 규칙을 순서대로 적용하는 것: ok
      - 최소 구성은 이렇습니다.
          - 가격 데이터 로딩: ok
          - 전략 신호 생성: 이해안됨: ok
          - 주문 체결 시뮬레이션: ok
          - 수수료/세금 반영: nice to have이며 첫번째 버전에서는 필요없음 
          - 포트폴리오 가치 계산: ok
3. 데이터 품질을 가장 중요하게 보기
      - 수정주가 반영 여부: ok
      - 액면분할/배당 처리: ok
      - 상장폐지 종목 포함 여부: 필요없음 
      - 생존자 편향 방지: 필요없음
      - 분봉이면 틱/체결 지연도 고려: 필요없음 
4. 검증 지표를 함께 만들기
      - 누적수익률 : ok
      - 연복리수익률(CAGR) : ok 
      - MDD : ok
      - 승률 : 필요없음 
      - 손익비 : 필요없음 
      - 샤프지수 : 필요없음 
      - 거래 횟수 : 필요없음 
      - 포지션별 평균 보유기간 : ok 
5. 과최적화 방지 장치 넣기 
      - 학습 구간 / 검증 구간 / 테스트 구간 분리
      - 워크포워드 테스트
      - 파라미터를 너무 많이 두지 않기
      - 전략이 여러 종목, 여러 기간에서 재현되는지 보기

========================================
구현방향 

1. backtester/ 폴더와 Python 기본 구조 생성
2. CSV 기반 일봉 데이터 입력 형식 정의 
   - CSV 형식은 Date, Open, High, Low, Close, Adj Close, Volume
3. 주식가격 정보(CVS)는 외부 검색을 통해 직접 구할것.
   - 주로 yahoo finance나 google finance 또는 investing.com에서 구할 수 있으며, 가장 적절한 source인지 확인하고 선정할것.
   - 구해진 가격 정보는 backtester/data/ 폴더 아래에 넣을것 
   - 주식가격정보 파일이 있고 그 파일의 내용이 backtest하려는 기간내 모두 존재하는 경우, 새로 정보를 구하지 않고 그  파일을 사용. 없는 경우에만 새로 정보를 구할것.
4. 종목은 "종목후보" 에 있는 종목을 사용하며, 종목을 추가할 경우, "종목후보" 부분에 추가로 작성할것 
5. 전략은 "전략후보" 에 있는 전략을 사용하며, 전략을 추가할 경우, "전략후보" 부분에 추가로 작성할것 
6. 이동평균 전략 1개 구현 (첫번쨰 전략은 추세추종)
   - 상세한 전략내용은 strategy.md 파일을 만들어서 내용을 작성할것
   - 여러개의 전략이 있으므로, 각각 구분하여 정리할것 
7. CLI로 백테스트 실행 가능하게 구성.
   - 종목 선택 및 기간을 정할 수 있어야 함.
   - 투자금액 설정가능해야 함. 초기 투자금(필수) 과 매년 연초 (1월1일) 추가투자금(선택)
   - 투자자산이 1개가 아닌경우, 비율이 설정되었으면 그 비율만큼 매수하고 비율이 설정되어있지 않으면 동일비중으로 매수할것 
8. 성과표 출력
   - 타이틀을 붙일것 MY BACKTESTING REPORT 정도 ?
   - 사용한 전략 이름과 상세내용을 보여줄것 
   - 매해 성과 (매년 연말기준 포트폴리오의 수익율, MDD등) 표시 
   - 최종 성과 (백테스트 기간으로 선정한 마지막날) 도 표시할것 
   - 비교를 위한 벤치마크  성과 표시 (매해 성과 및 최종성과)
       > 종목이 1개인 경우에는 그 종목이 벤치마크가 됨 
       > 종목이 N개인 경우에는 VOO가 벤치마크가 됨 
   - Format: 보여주는 성과표는 다음과 같이 할것 
[TITLE]
Strategy; 전략정보 표
Yearly Performance -> 매년 성과를 표기
year | end_value  |  return  |   mdd   | BM value   | BM return | BM mdd   |

Final Performance -> 최종 성과를 표기
period:               2018-01-02 -> 2026-04-29
ending_value:         $30,227.67
cumulative_return:    202.28%
CAGR:                 14.22%
overall_MDD:          -21.88%
average_holding_days: 74.4
- 총 투자원금 (첫해 투자금 및 매해 추가 투자금 총합)

=> 각 데이터에 적절한 white space를 두어 테이블의 칸이 삐뚤삐뚤하지 않도록 만들것 

9. 차트는 나중에 구현 (첫 버전에서는 제외) 

========================================
종목후보 
1) 주식 자산 
- VOO, SPY, SSO, SPXL
- QQQ, QLD, TQQQ
- EEM
2) 채권 자산 
- SHY, IEF, TLT
3) 기타 자산 
- GLD, SLV, COPX

전략후보 
- 듀얼 모멘텀전략: 상반된 경향이 있는 주식 자산을 2개 선정 (e.g. EEM 과 VOO 또는 VOO 와 QQQ) 하며, 적정기간 (120일 200일)의 수익율이 채권 (3년 또는 10년) 보다 높을 경우 높은 자산 1개에 투자. 만약 2개 자산 모두 채권보다 낮을 경우에는 채권 보유. 만약 주식자산과 채권 모두 마이너스라면 현금을 보유. 매달말에 한번씩 리밸런싱
- 추세추종 전략: 50일, 200일 등 주요 이동평균선의 정배열 상태를 상승 추세로 판단하여 매수하고, 이탈 시 매도
- 자산배분 전략: 주식자산 중 1개, 채권자산 중 1개, 기타자산 중 1개를 선정하여 각각 비율로 매수하고, 적정기간 (120일, 200일, 매년말)에 비중이 늘어난곳을 줄이고, 줄어든곳을 늘려서 처음에 설정한 비율로 맞춤.
